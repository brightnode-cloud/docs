Create multiple Bnodes at once with identical configurations (useful for parallel workloads or distributed training).

<RequestExample>
```sh Command
brightnodectl create bnodes [flags]
```
</RequestExample>

## Example

Create 3 identical Bnodes with the name "training-worker" in the Secure Cloud:

```sh
brightnodectl create bnodes \
  --name "training-worker" \
  --bnodeCount 3 \
  --gpuType "NVIDIA GeForce RTX 3090" \
  --gpuCount 1 \
  --secureCloud \
  --imageName "Brightnode/pytorch:2.0.1-py3.10-cuda11.8.0-devel"
```

## Flags

<ResponseField name="--name" type="string">
A custom name for the Bnodes. All Bnodes in the group will share this base name.
</ResponseField>

<ResponseField name="--bnodeCount" type="integer" default={1}>
The number of Bnodes to create.
</ResponseField>

<ResponseField name="--gpuType" type="string">
The GPU type to use for the Bnodes (e.g., `NVIDIA GeForce RTX 4090`, `NVIDIA B200`, `NVIDIA L40S`). Use the GPU ID (long form) from the [GPU types reference](/references/gpu-types) table to specify the GPU type.
</ResponseField>

<ResponseField name="--gpuCount" type="integer" default={1}>
The number of GPUs to allocate to each Bnode.
</ResponseField>

<ResponseField name="--secureCloud">
Create the Bnodes in the Secure Cloud tier, which offers enterprise-grade infrastructure with enhanced reliability.
</ResponseField>

<ResponseField name="--communityCloud">
Create the Bnodes in the Community Cloud tier, which typically offers lower pricing with spot instance availability.
</ResponseField>

<ResponseField name="--imageName" type="string">
The Docker container image to use for the Bnodes (e.g., `Brightnode/pytorch:latest`).
</ResponseField>

<ResponseField name="--templateId" type="string">
The ID of a template to use for Bnode configuration, which pre-defines the image and environment settings.
</ResponseField>

<ResponseField name="--containerDiskSize" type="integer" default={20}>
The size of the container disk in gigabytes for each Bnode.
</ResponseField>

<ResponseField name="--volumeSize" type="integer" default={1}>
The size of the persistent volume in gigabytes for each Bnode.
</ResponseField>

<ResponseField name="--volumePath" type="string" default="/workspace">
The mount path for the persistent volume inside each container.
</ResponseField>

<ResponseField name="--networkVolumeId" type="string">
The ID of an existing network volume to attach to all Bnodes for shared storage.
</ResponseField>

<ResponseField name="--cost" type="float">
The maximum price ceiling in dollars per hour. If not specified, Bnodes will be created at the lowest available price.
</ResponseField>

<ResponseField name="--mem" type="integer" default={20}>
The minimum system memory required in gigabytes for each Bnode.
</ResponseField>

<ResponseField name="--vcpu" type="integer" default={1}>
The minimum number of vCPUs required for each Bnode.
</ResponseField>

<ResponseField name="--env" type="string">
Environment variables to set in the containers. Specify multiple times for multiple variables.
</ResponseField>

<ResponseField name="--args" type="string">
Additional arguments to pass to the containers when they start.
</ResponseField>

<ResponseField name="--ports" type="string">
Ports to expose from the containers. Maximum of 1 HTTP port and 1 TCP port allowed per Bnode.
</ResponseField>
